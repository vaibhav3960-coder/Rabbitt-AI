# FireReach - Autonomous Outreach Engine

FireReach is a lightweight, single-feature autonomous outreach prototype for SDRs/GTM teams. The agent takes an Ideal Customer Profile (ICP) and task description, captures live signals, reasons through the data to generate an account brief, and automatically sends a hyper-personalized email.

## Setup Instructions
1. Navigate to `backend/` and create a `.env` file with your `GEMINI_API_KEY`.
2. Start the backend: `cd backend && source venv/bin/activate && uvicorn main:app --reload`
3. Start the frontend: `cd frontend && npm run dev`
4. Open your browser to the local Vite URL.

## Logic Flow
The agent follows a Sequential Reasoning flow enforced by the system prompt:
1. **Signal Capture:** The user inputs an ICP and Task. The agent first calls `tool_signal_harvester` with a target company (or relies on the default) to fetch high-value intent triggers such as funding rounds or hiring trends.
2. **Contextual Research:** Once the agent has the signals, it calls `tool_research_analyst` to synthesize the signals against the user's ICP into an "Account Brief".
3. **Automated Delivery:** Happy with the research, the agent finally calls `tool_outreach_automated_sender`. This tool takes the brief, applies a strict Zero-Template Policy, and generates a highly personalized email implicitly tying the ICP's value proposition to the live buyer signals.

## Tool Schemas

### 1. `tool_signal_harvester` (Deterministic)
- **Description:** Fetches mock live data for a specific account. Captures triggers like funding rounds, hiring trends, and tech stack changes.
- **Arguments:** 
  - `target` (string): The name of the company to harvest signals on. If empty, the system provides a matching generic candidate.

### 2. `tool_research_analyst` (Account Insight)
- **Description:** Takes the harvested signals + user ICP to generate a 2-paragraph Account Brief highlighting specific pain points and strategic alignment.
- **Arguments:**
  - `signals` (string): The json payload returned by the harvester.
  - `icp` (string): The user's ideal customer profile statement.

### 3. `tool_outreach_automated_sender` (Execution)
- **Description:** Transforms the research into a hyper-personalized email and automatically simulates dispatching it using a mail service.
- **Arguments:**
  - `brief` (string): The account brief generated by the research tool.
  - `target_email` (string): The recipient email.

## System Prompt
```
You are the FireReach Autonomous Outreach Engine.
You operate on a Zero-Template Policy. 
Follow sequential reasoning: 
1. Capture Live Signals (tool_signal_harvester) based on the user's task.
2. Contextual Research (tool_research_analyst): Analyze the signals and user's ICP.
3. Automated Delivery (tool_outreach_automated_sender): Draft and send the final email.
You MUST rely on your tools to capture data, research, and send the email.
Once the email is sent by the tool, return a summary of what you did.
```
